# Workflow разработки xyliganimbot

Правила выполнения работ по `docs/tasklist_v_<номер_релиза>.md` для разработки бота согласно `docs/vision.md` и `docs/conventions.md`.

## Общий принцип

Работа ведётся **строго по плану** из `docs/tasklist_v_<номер_релиза>.md` последовательно, итерация за итерацией. Каждая итерация выполняется полностью перед переходом к следующей.

## Процесс работы над итерацией

### 1. Начало итерации

- Выбрать следующую незавершённую итерацию из `docs/tasklist_v_<номер_релиза>.md`.
- Изучить задачи в рамках выбранной итерации.

### 2. Планирование и согласование

**Перед реализацией** необходимо:
- Предложить предполагаемое решение с **отрезками кода** (структура файлов, основные функции, примеры).
- Показать, как решение соответствует `docs/vision.md` и `docs/conventions.md`.
- Дождаться **согласования** от пользователя.

### 3. Реализация

**После согласования**:
- Реализовать решение согласно согласованному плану.
- Следовать принципам из `docs/conventions.md` (KISS, простые функции, стандартный logging и т.д.).

### 4. Проверка и подтверждение

- Показать результат реализации.
- Дождаться **подтверждения** выполнения от пользователя.

### 5. Обновление прогресса

После подтверждения:
- Обновить таблицу прогресса в `docs/tasklist_v_<номер_релиза>.md` (статус итерации).
- Отметить выполненные задачи (чекбоксы `[x]`).
- Добавить примечания при необходимости.

### 6. Коммит

- Сделать коммит в репозиторий с понятным сообщением (например: "Итерация 1: каркас бота, конфиг, healthcheck").
- Сделать коммит в онлайн репозиторий https://github.com/kretmax-creator/xyliganimbot с комментарием на английском.

### 7. Переход к следующей итерации

**Перед переходом к следующей итерации**:
- Согласовать завершение текущей итерации.
- Получить подтверждение для начала следующей.

## Важные правила

- ❌ **Не переходить** к следующей итерации без завершения текущей.
- ❌ **Не реализовывать** код без предварительного согласования решения.
- ✅ **Всегда согласовывать** изменения перед коммитом.
- ✅ **Обновлять прогресс** после каждой завершённой задачи.

## Пример последовательности для одной задачи

1. "Начинаю работу над итерацией 1. Предлагаю структуру: `src/bot.py`, `src/config.py`, ..."
2. [Пользователь согласовывает]
3. "Реализую согласно плану..."
4. "Готово. Результат: ..."
5. [Пользователь подтверждает]
6. "Обновляю прогресс в tasklist_v_<номер_релиза>.md"
7. "Делаю коммит: 'Итерация 1: каркас бота...'"
8. "Итерация 1 завершена. Переходить к итерации 2?"
